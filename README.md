# Remote Photo Slides

집 PC의 사진 폴더를 같은 Wi-Fi의 PC, 태블릿, 휴대폰에서 선택하고 랜덤 슬라이드쇼로 재생하는 로컬 서버입니다.

## 기본 설정

- 사진 최상위 폴더: `D:\민욱\사진`
- 서버 주소: `http://localhost:8080`
- 인식 확장자: `.jpg`, `.jpeg` (대소문자 구분 없음)
- 선택된 폴더의 하위 폴더까지 자동 검색
- 사진 목록은 30초마다 자동 확인
- 원본은 변경하지 않고 최대 1920×1080 재생용 사본을 서버 캐시에 생성
- 사진 EXIF의 GPS를 한 번만 분석해 PC에 저장
- 무료 GeoNames 데이터로 가까운 도시 표시
- 무료 geoBoundaries 경계 자료로 실제 행정구역 내부 판정
- 위치 문구를 누르면 API 키 없이 Google 지도에서 좌표 열기

## Google Places 랜드마크 (선택)

관리 화면에서 Google Places API 키를 저장하면 사진에서 30m 이내인 장소를 우선 표시하고, 없으면 300m 이내에서 Google 인기순 장소를 표시합니다. 폴더명에는 별도 점수를 주지 않습니다. API 키와 검색 캐시는 GitHub에 올라가지 않는 `data` 폴더에만 저장됩니다.

- 가까운 GPS 사진을 약 250~300m 지역으로 묶어 거리순과 인기순을 각각 한 번 검색
- 슬라이드쇼 재생 중에는 Google API를 호출하지 않음
- 월 4,000회에서 애플리케이션이 자동으로 검색 중지
- 30m 이내는 거리순, 그보다 멀면 300m 이내 Google 인기순으로 선택
- 제조업체, 회사, 창고, 도매업체 등 일반적인 방문 장소가 아닌 후보는 제외
- 이름에 `주식회사`나 `(주)`가 들어간 후보는 제외
- 두 기준에 맞는 Google 장소가 없으면 행정구역만 표시
- Google이 한국어 이름을 제공하지 않은 장소와 지역은 표시하지 않고 한국어 국가명으로 대체
- Google 장소 표시 정보는 29일 후 갱신하고 Place ID는 사진 위치 캐시에 보관

Google Cloud에서 Places API (New)를 활성화하고 결제 계정을 연결한 뒤 관리 화면에 키를 입력합니다. 키는 Places API로만 제한하고 Google Cloud에도 별도의 사용량 알림과 할당량을 설정하는 것을 권장합니다.

## 개인 장소

집처럼 항상 같은 이름으로 표시할 장소는 GitHub에 올라가지 않는 `data/locations/private-places.json`에 저장합니다.

```json
{
  "places": [
    {
      "name": "우리집",
      "latitude": 0,
      "longitude": 0,
      "radiusMeters": 150,
      "city": "지역명",
      "cityRadiusMeters": 300
    }
  ]
}
```

개인 장소는 Google 검색 결과보다 우선하며 거리 문구 없이 지정한 이름만 표시됩니다.
`cityRadiusMeters`를 지정하면 개인 장소 이름의 반경과 별도로 주변 지역명을 보정할 수 있습니다.

## 실행

처음 한 번:

```powershell
npm install
```

이후에는 `start.bat`을 실행합니다. 화면에 표시되는 `같은 Wi-Fi에서 열기` 주소를 갤럭시 탭에서 열면 됩니다.

이 PC에서는 Windows 로그인 시 `start-hidden.vbs`를 통해 서버가 자동으로 실행되도록 시작 프로그램에 등록할 수 있습니다. 자동 실행 상태에서도 명령창은 표시되지 않습니다.

PC에서 사진 폴더를 추가하거나 삭제한 뒤 별도 저장 버튼을 누를 필요는 없습니다. 이미 선택된 폴더 내부의 JPG 변경은 자동 반영됩니다. 선택할 폴더 자체를 바꿀 때만 관리 화면에서 `선택 저장`을 누릅니다.

## 선택 설정

다른 경로나 포트를 사용하려면 실행 전에 환경 변수를 지정할 수 있습니다.

```powershell
$env:PHOTO_ROOT='D:\사진'
$env:PORT='8080'
npm start
```
